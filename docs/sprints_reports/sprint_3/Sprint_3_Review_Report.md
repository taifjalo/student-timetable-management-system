# Team 6 - Sprint 3 Review Report

## Sprint Goal

The goal of Sprint 3 was to extend the MVP with core features (login, messaging, UI modals), strengthen automated testing, establish CI/CD via Jenkins, prepare the application for functional review, and create the first Docker image.

## Completed User Stories / Tasks

The full Product Backlog and Sprint setup are available in the Trello workspace:
[Trello Workspace – Student Timetable Management System](https://trello.com/b/is0nq5uS/sprint-3)

- Login system implemented and tested
- Messaging feature completed
- Modals and UI improvements implemented
- MVP now usable end‑to‑end
- Jenkins CI/CD pipeline integrated (checkout → build → tests → coverage)
- JaCoCo coverage report generated
- Expanded unit test suite, coverage improved
- All main features validated for functional review
- Initial Dockerfile created, local image built and runs successfully
- Sprint goals were documented in Trello
- The Sprint 3 planning report was completed and pushed to GitHub

## Demo Summary (What Was Shown in the Review)

During the Sprint Review, the team demonstrated:

- Logging in and navigating through the system
- Sending and receiving messages
- Using UI modals
- Running Jenkins build with unit tests & coverage
- Displaying JaCoCo HTML coverage report
- Starting the application via Docker locally

## What Went Well

- CI/CD pipeline greatly improved development speed
- MVP features were completed earlier than expected
- Team collaboration and task sharing was effective
- Docker image worked on the first iteration
- Effective use of Trello for task organization

## What Could Be Improved

- Code coverage could still be increased
- UI still needs polish and error-handling improvements
- Some tests required refactoring due to dependency issues
- Pipeline could include automated linting in next sprint

## Next Sprint Focus

- Extend Docker support (multi‑stage build & production image)
- Improve UI/UX and error handling
- Increase test coverage and add integration tests
- Expand Jenkins pipeline (linting, notifications, static analysis)
- Planning for Sprint 4
- Trello update
