# Team 6 - Sprint 6 Review Report

## Sprint Goal

The goal of Sprint 6 was to extend localization support into the database layer and to improve overall code quality and maintainability. The sprint also focused on preparing the project for acceptance by refactoring, planning acceptance tests, and performing systematic code review via SonarQube.

## Completed Tasks

The full Product Backlog and Sprint setup are available in the Trello workspace:
[Trello Workspace – Student Timetable Management System](https://trello.com/w/sep1_studenttimetablemanagementsystem/home)

* Database was localized using separate translation tables for courses, groups, and notifications: (COURSE_TRANSLATIONS, GROUP_TRANSLATIONS, NOTIFICATION_TRANSLATIONS)
    + Database localization plan was created
    + ER diagram and UML models were updated to reflect the database localization changes and overall architecture.
* Static code analysis tools such as PMD, SonarQube, and SpotBugs were used to identify and fix code smells.
    + A Statistical Code Review Report was created, documenting code metrics and identified issues.
    + Refactoring was performed to simplify complex methods, remove redundant code, and improve naming conventions.
* Acceptance test cases were designed covering functional, usability, and performance testing.
* Initial design was documented in GitHub inside the `/docs` folder
* Team meetings were held twice per week to discuss the project.
* Sprint 6 Planning Report was written and uploaded to the GitHub repository.
